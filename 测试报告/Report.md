## 1. 测试概述

### 1.1 测试对象
- **设备**：香橙派 [型号：Orange Pi 3B]  
- **操作系统**：Linux 5.4.125 [Ubuntu 22.04.5]  
- **穿透工具**：[cpolar 3.0]  

### 1.2 访问地址
- **公网地址**：`http[s]://295cbd01.r16.cpolar.top`  
- **内网地址**：`http://192.168.0.114:81`  

### 1.3 测试范围
- **功能测试**：公网HTTP/HTTPS访问，路径/参数传递，表单提交等基础功能。  
- **可靠性测试**：穿透服务重启、网络波动、设备重启后的自愈能力。  
- **性能测试**：响应时间、并发访问、带宽占用。  
- **安全测试**：端口暴露、未授权访问防护、敏感路径控制。  

### 1.4 不测试的内容
- 网站内部的业务逻辑。  
- 穿透工具本身的源码安全审计。  

### 1.5 测试环境
| 角色               | 硬件/软件配置                          | 网络环境          |
| ------------------ | -------------------------------------- | ---------------- |
| 香橙派（服务端）   | CPU: RK3588, RAM 4GB, Ubuntu          | 内网 192.168.0.114 |

### 3.2 可靠性测试（RT）

#### RT-01：穿透服务重启后恢复
| 项目           | 内容                                                         |
| -------------- | ------------------------------------------------------------ |
| **用例编号**   | RT-01                                                        |

| **预期结果**   | 重启后公网地址可重新访问，连续失败次数不超过2次（即短暂中断后恢复）。 |

## 6. 测试总结

### 6.1 测试结论
- **总用例数**：6  
- **通过数**：5  
- **失败数**：1   
- **通过率**：83.33%  

### 6.2 主要风险与建议
- **风险1**：家庭宽带上行带宽波动可能导致高峰期访问缓慢。  
  **建议**：限制单IP最大并发连接，或升级宽带。  
- **风险2**：CPU性能弱，在运行其他家庭服务时，导致没有足够算力分配。  
  **建议**：升级CPU。
